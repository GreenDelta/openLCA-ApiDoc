Result

A `Result` message is basically just a reference to a real result on the

server side. As result data can be really large, we do not add them directly

to the result. Instead you call the respective methods of the result service

to get them. Because of this, you need to explicitly call `Dispose` when you

do not need a result anymore. Otherwise you will create memory leaks as the

result still exists on the server side.

GetImpactFactors ImpactFactorRequest ImpactFactorResponse stream

Get the used impact factors of a result. The request can be for: * an indicator: returns the non-zero factors of that indicator * a flow: returns the factors of that flow for all indicators (including zero values) * an indicator and flow: returns the factor for that flow which may be zero Nothing is returned if no indicator and flow is given or if the result does not have an impact assessment result.
